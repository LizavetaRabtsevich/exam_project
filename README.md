Данный проект представляет собой REST API для управления списком задач (To-Do List). API реализует полный набор CRUD-операций для ресурса Task и позволяет создавать, получать, обновлять и удалять задачи с использованием стандартных HTTP-методов GET, POST, PUT, PATCH и DELETE. Все данные передаются и принимаются в формате JSON, а API использует корректные HTTP-коды состояния, такие как 200 OK, 201 Created и 404 Not Found.

Каждая задача содержит уникальный идентификатор, название, описание и статус выполнения. В качестве возможных статусов используются значения todo, in_progress и done. Реализованы эндпоинты для получения списка всех задач (/tasks), получения одной задачи по идентификатору (/tasks/{id}), создания новой задачи, полного обновления задачи, частичного обновления (PATCH) и удаления задачи.

Проект реализован на Java с использованием Spring Boot. Для хранения данных используется реляционная база данных SQLite, доступ к которой осуществляется через Spring Data JPA и Hibernate. Данные сохраняются на диске и не теряются при перезапуске приложения. Для повышения производительности реализовано кеширование с использованием Spring Cache, что позволяет избегать лишних обращений к базе данных при повторных запросах.

В проекте реализована система безопасности на базе Spring Security и JWT-аутентификации. Доступ к защищённым эндпоинтам осуществляется только после успешной аутентификации пользователя. В роли упрощённого API Gateway используются servlet-фильтры, которые обеспечивают логирование входящих запросов и ответов, а также реализуют ограничение трафика (rate limiting).

Для мониторинга состояния приложения и анализа его работы подключены Spring Actuator и Micrometer с поддержкой Prometheus. Приложение предоставляет метрики производительности, использования памяти, количества запросов и времени их обработки, а также эндпоинты состояния (health checks).

Для повышения отказоустойчивости реализованы механизмы Retry и Circuit Breaker с использованием Resilience4j. В случае временной недоступности базы данных или ошибок при обращении к ней сервис автоматически выполняет повторные попытки выполнения операции, а при превышении порога ошибок переключается на fallback-логику, предотвращая каскадные сбои.

Проект имеет модульную архитектуру с разделением на контроллеры, сервисы, репозитории и конфигурационные классы, а также покрыт unit-тестами для контроллеров с использованием MockMvc и Mockito. Реализация демонстрирует не только базовую работу REST API, но и применение ключевых backend-паттернов, используемых в реальных production-приложениях.
